# Comet-AI v0.2.6 (2026-03-30)

## Highlights
- **Hardened Mobile Sync**: Implemented reliable auto-reconnection for mobile devices using `shared_preferences`. The mobile app now remembers the last connected desktop and recovers connections automatically.
- **Auto-Relay for Generated Files**: AI-generated reports (PDF/DOCX/PPTX) are now automatically synced to the local relay directory and instantly pushed to the mobile app for immediate viewing.
- **Firebase Persistence Hygiene**: Automated background routines now periodically purge orphaned relay files from Firebase Storage and clean up old `temp_` database entries after 24 hours.
- **Unified version source**: UI reads the app version directly from `package.json`/Electron, so Welcome, Settings, and badges stay in sync automatically.
- **DOCX/PPTX/PDF improvements**: richer formatting, inline images, tables, palettes, and safer DOCX header/footer handling.
- **Robust CREATE_FILE_JSON parsing**: accepts wrapped commands, slide aliases, mixed image locations; auto-selects PPTX when slides are provided.
- **BrowserView resize fixes**: Sidebar position changes and fullscreen toggle now instantly recalculate and apply BrowserView bounds without delay.
- **DOCX page configuration**: Explicit US Letter page size (12240x15840 DXA), margins, and orientation now set for maximum compatibility with Microsoft Word.

## Fixes
- **IPC Scoping Resolution**: Fixed a `ReferenceError` in `main.js` within the `desktop-control` handler where arguments were incorrectly scoped, causing crashes during remote chat.
- **CloudSync Initialization**: Resolved property access errors where `firebaseService` was expected to have a `database` member; now correctly uses `getDatabase` for real-time sync.
- **DOCX generation crash**: Resolved `reading 'children'` error by implementing valid Header/Footer construction and JS syntax cleanup.
- **PPTX watermarking**: Removed cover watermark for PPTX thumbnails; watermark remains on inner slides only.
- **BrowserView Resizing**: Fixed layout issues where the BrowserView failed to resize when the sidebar was moved or fullscreen was toggled.
