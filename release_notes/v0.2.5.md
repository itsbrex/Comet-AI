# Comet AI Browser v0.2.5-stable (2026-03-26)

## Highlights
- **Automation & scheduling finesse** – The automation settings page now exposes active tasks, a “+ Create Task” quick action, and Raycast-native endpoints that allow the AI to list or delete automations directly. The scheduling modal respects JSON task payloads and keeps the panel scrollable even when long.
- **macOS-native polish** – Native App menus (Comet AI, File, Edit, View, Window, Help) ship with contextual spacing and descriptions, and the mac build now uses a SwiftUI-only translucent AI chat sidebar that delivers an Apple-like experience.
- **Human-in-the-loop flow** – Low/mid-risk commands that aren’t auto-approved show the existing QR verification modal instead of blocking, giving the AI a way to request approvals while surfacing the current automation state for inspection/removal.
- **Charm setup & theme refresh** – The welcome/setup guide features the new Charm light theme, lets you choose a theme during onboarding, auto-detects your timezone, and doesn’t exit until required steps (including API key configuration) are completed.
- **Panel & UX polish** – Download, clipboard, cart, and ecosystem overlays now close when you click outside, and the entire UI reflects the Charm aesthetic while keeping automation dialogs within viewable heights on macOS.

## Fixes
- Removed duplicate `SpotlightSearchOverlay` import and resolved the JSX parsing error that was preventing the scheduling modal from rendering.
- Prevented `automation:get-tasks` errors when the automation service isn’t ready and fixed the `ClientOnlyPage` `useEffect` loop.
- Ensured download/clipboard/autop panels disable the browser view when open so background rendering can’t bleed through.

## Notes
- Raycast extension callers can now hit the new automation list/delete IPC hooks and surface their UI confirmation state for approvals.
- This version continues to enforce the triple-lock security model and background scheduling service introduced in prior releases while improving UI polish for Automation, Raycast, and macOS builds.
